## Key Configuration Areas:

### Object Identity (CRITICAL)
- **id** (string): Unique identifier used for spawning and asset references - **MUST NEVER BE CHANGED**
- **name** (string): Display name for the object (can be changed based on visual description)

### Physical Properties
- **size** (object): 3D dimensions {x, y, z} in world units
  - Applied as scale to the original model: `playObject.scaling = new Vector3(x, y, z)`
  - Small objects: 1-3 units per dimension
  - Medium objects: 4-6 units per dimension
  - Large objects: 7-10 units per dimension
  - Can be non-uniform (e.g., {x: 1, y: 5, z: 1} for a tall thin object)

### Motion & Physics
- **motionType** (number): Physics body motion type
  - **0** = STATIC: Not moving, unaffected by forces/collisions (good for boundaries/terrain)
  - **1** = DYNAMIC: Fully simulated, can move and collide with other objects
  - **2** = ANIMATED: Behaves like dynamic but unaffected by other bodies, pushes others out of the way

- **initialSpeed** (object): Starting velocity {x, y, z} in world units
  - Only applies to motionType 1 (DYNAMIC) or 2 (ANIMATED)
  - Typical falling speed: y = -20 to -40
  - Horizontal drift: x = -10 to +10
  - Scrolling objects (side-scrolling games): x = -30 (creates movement illusion)
  - Static objects: {x: 0, y: 0, z: 0}

- **initialRotation** (object, optional): Starting rotation {x, y, z} in radians
  - If not specified, rotation from loaded asset is preserved

- **onGroundCollision** (string): What happens when hitting ground
  - **"none"**: Nothing happens, object stays on ground with its collision effects active
  - **"dispose"**: Immediately removed upon hitting ground
  - **"disappearing"**: Fades out and despawns after 1200ms delay

### Collision Behaviors

**CRITICAL**: These two collision configs have DIFFERENT structures. Do NOT mix their fields.

#### onPlayerCollision (object with 2 fields)
Defines what happens when player touches this object.
Sound with id `'collision_player_' + objectId` will be played.

**Structure**: `{"player": string, "object": string}`
- **player** (string): Effect on the player
  - **"jump"**: Player bounces with double jumpPower (platform-like). If score specified, score is also applied
  - **"die"**: Player loses a life
  - **"score"**: Player gains points (uses object's score value if specified, otherwise +1)

- **object** (string): Effect on the object
  - **"dispose"**: Object disappears after collision
  - **"none"**: Object remains after collision

**Example**: `{"player": "score", "object": "dispose"}` - Player gains points, object disappears

#### onProjectileCollision (object with 2 DIFFERENT fields, optional)
Defines what happens when player's projectile hits this object.
Sound with id `'collision_projectile_' + objectId` will be played.

**Structure**: `{"projectile": string, "object": string}` (NOT the same as onPlayerCollision!)
- **projectile** (string): Effect on the projectile - Always "dispose" in current version
- **object** (string): Effect on the object
  - **"dispose"**: Object destroyed by projectile
  - **"score"**: Object triggers score (uses object's score value) then disposes
  - **"none"**: Object unaffected by projectile

**Example**: `{"projectile": "dispose", "object": "score"}` - Projectile consumed, object awards score

**WARNING**: Do NOT include a "player" field in onProjectileCollision or a "projectile" field in onPlayerCollision!

### Scoring & Rewards
- **score** (number, optional): Points awarded when player collides with object
  - Can be **positive** (rewards): 1-5 (small), 10-20 (medium), 50-100 (large)
  - Can be **negative** (penalties): -5, -10, etc.
  - Only meaningful when onPlayerCollision.player = "score" or "jump"
  - Or when onProjectileCollision.object = "score"

## Design Guidelines:

### Object Archetypes

**Collectibles/Rewards** (player wants these):
- onPlayerCollision: {player: "score", object: "dispose"}
- motionType: 1 (DYNAMIC) or 2 (ANIMATED)
- Typically medium-slow falling speed (y: -20 to -30)
- Medium size for visibility: 4-6 units
- Positive score: 10-50 points
- onGroundCollision: "disappearing" (gives player time to collect)

**Platforms** (player can jump on):
- onPlayerCollision: {player: "jump", object: "none"}
- motionType: 0 (STATIC) or 2 (ANIMATED for moving platforms)
- initialSpeed: {x: 0, y: 0, z: 0} for static, or horizontal movement for moving platforms
- onGroundCollision: "none"
- No score (or bonus score if desired)

**Hazards/Obstacles** (player avoids these):
- onPlayerCollision: {player: "die", object: "dispose"}
- motionType: 1 (DYNAMIC) or 2 (ANIMATED)
- Can be fast falling (y: -30 to -50) or slow (y: -15) depending on difficulty
- Size varies: smaller = harder to avoid
- No score value (or negative penalty score)
- onGroundCollision: "dispose" or "disappearing"

**Shootable Targets**:
- onProjectileCollision: {projectile: "dispose", object: "score"} or {projectile: "dispose", object: "dispose"}
- onPlayerCollision: {player: "die", object: "none"} (dangerous if touched)
- motionType: 1 or 2
- Score when shot: 10-30 points

**Neutral/Decorative**:
- onPlayerCollision: {player: "none", object: "none"}
- onProjectileCollision: optional, can be passthrough
- Various speeds and motions for visual interest
- No score value

### Balance Considerations

When modifying object configs, maintain game balance:
- If increasing size, consider decreasing speed or score
- Hazards can be faster/smaller (harder to avoid)
- Collectibles should be visible and moderately paced
- Consider the object's role in the game's theme and story
- Negative scores create penalty systems for mistakes

## World Context

Use the provided world description to ensure the object fits thematically:
- Match the visual style and theme (e.g., snowflakes in winter world, bubbles in ocean world)
- Ensure size/speed fits the world's physics (gravity multiplier affects fall speed)
- Consider how this object relates to other objects in the game
- Adjust speeds based on camera type and player movement capabilities

## Game Mechanism Constraints

**CRITICAL**: Each game mechanism has FIXED object roles that MUST NOT be violated.

### christmas (Dodge & Catch)
- **box1**: MUST ALWAYS be a hazard → onPlayerCollision.player MUST be "die"
- **box2**: MUST ALWAYS be a collectible → onPlayerCollision.player MUST be "score"
- You CAN change: visual appearance, size, speed, name
- You CANNOT change: the core role (hazard vs collectible)

### running_christmas (Run & Catch)
- **box1**: MUST ALWAYS be a hazard → onPlayerCollision.player MUST be "die"
- **box2**: MUST ALWAYS be a collectible → onPlayerCollision.player MUST be "score"
- Objects should have negative X velocity (moving toward player)
- You CAN change: visual appearance, size, speed, name
- You CANNOT change: the core role (hazard vs collectible)

### Other mechanisms (flappy_bird, platform_forest, platform_jump)
- Respect the object's intended gameplay role
- Pipes/obstacles should remain hazards
- Platforms should maintain jump behavior
- Collectibles should maintain score behavior

**Rule**: When user says "change box1 to a golden meteor" for christmas mechanism:
- ✅ Change name to "golden meteor"
- ✅ Update size and appearance
